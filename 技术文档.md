### 1.3 前后端通信

- **Wails** 通过 `window.go.main.App` 暴露 Go 方法给前端
- 前端通过 `go.main.App.MethodName(...)` 调用
- 菜单事件通过 `runtime.EventsOn` / `runtime.EventsEmit` 传递

## 二、数据模型

### 2.1 实体 (Entity)

```go
type Entity struct {
    ID         string     `json:"id"`         // UUID
    TypeID     string     `json:"typeId"`     // 类型 ID
    Label      string     `json:"label"`      // 显示标签
    Properties []Property `json:"properties"` // 自定义属性
    X          float64    `json:"x,omitempty"`
    Y          float64    `json:"y,omitempty"`
}
```

### 2.2 关系 (Relationship)

```go
type Relationship struct {
    ID         string     `json:"id"`
    Source     string     `json:"source"`     // 源实体 ID
    Target     string     `json:"target"`     // 目标实体 ID
    TypeID     string     `json:"typeId"`
    Label      string     `json:"label"`
    Properties []Property `json:"properties"`
    Directed   bool       `json:"directed"`   // 是否Directed
}
```

### 2.3 实体类型 / 关系类型

```go
type EntityType struct {
    ID    string `json:"id"`
    Name  string `json:"name"`
    Color string `json:"color"`
    Icon  string `json:"icon"`
}

type RelationType struct {
    ID    string `json:"id"`
    Name  string `json:"name"`
    Color string `json:"color"`
}
```

### 2.4 图数据 (GraphData)

```go
type GraphData struct {
    Entities      []Entity       `json:"entities"`
    Relationships []Relationship `json:"relationships"`
    EntityTypes   []EntityType   `json:"entityTypes"`
    RelationTypes []RelationType `json:"relationTypes"`
}
```

## 四、分析算法

### 4.1 链接分析 (Link Analysis)

- **算法**：BFS（广度优先搜索）
- **输入**：实体 ID，深度 N
- **输出**：N 跳内可达的实体与关系
- **复杂度**：O(V + E)

### 4.2 路径分析 (Path Analysis)

- **算法**：BFS 最短路径
- **输入**：源实体 ID，目标实体 ID
- **输出**：所有最短路径（节点序列）
- **说明**：若有多条等长最短路径，全部返回

### 4.3 群集分析 (Cluster Analysis)

- **算法**：标签传播 (Label Propagation)
- **迭代**：最多 100 轮
- **输出**：每个节点所属群集 ID
- **用途**：社区发现

### 4.4 社会网络分析 (SNA)

- **度中心性**：`degree / (n-1)`
- **介数中心性**：近似于经过该节点的最短路径占比
- **紧密中心性**：`reachable / totalDistance`

## 五、持久化

### 5.1 本地 JSON

- **路径**：`~/.relationship-analyzer/graph_data.json`
- **时机**：增删改后、应用关闭前
- **格式**：完整 `GraphData` JSON

### 5.2 NeuroDB

- **端口**：18839（内嵌默认）
- **协议**：TCP 文本协议
- **同步**：实体与关系通过 `CreateNode` / `CreateRelation` 同步
- **删除**：删除实体时调用 `DeleteNodeByUID`

## 六、前端状态管理

### 6.1 GraphContext

- **状态**：`entities`, `relationships`, `entityTypes`, `relationTypes`, `selectedNodeId`, `selectedEdgeId`, `layoutType`, `highlightIds`, `analysisResult`, `collapsedNodes`, `linkMode`, `linkSource`

- **Actions**：封装对 `window.go.main.App` 的调用

### 6.2 ThemeContext

- **模式**：`light` / `dark`
- **tokens**：主题色、边框、背景等

## 八、构建与运行

### 8.1 开发

```bash
# 后端 + 前端 dev server
wails dev

# 前端单独（连接 Wails dev）
cd frontend && npm run dev
```

### 8.2 构建

```bash
wails build
```

输出在 `build/bin/`：
- macOS: `relationship.app`
- Windows: `relationship.exe`

### 8.3 依赖

- Go 1.16+
- Node.js 18+
- Wails CLI v2
- 各平台构建工具（如 Xcode Command Line Tools、MSVC 等）
